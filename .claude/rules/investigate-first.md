# 버그 수정 프로토콜: 조사 먼저

버그를 수정할 때 바로 코드를 고치지 않고, 반드시 아래 단계를 따른다.

## Step 1: 데이터 흐름 추적

데이터가 어디서 시작해서 어디서 문제가 되는지 추적한다.

```
데이터 소스 (data.ts) → 변환 (transform.ts) → 스토어 (zustand) → 컴포넌트 → UI
```

## Step 2: 재현 시나리오 문서화

```markdown
- 전제 조건: (어떤 상태에서?)
- 트리거: (어떤 동작을 하면?)
- 기대 결과: (어떻게 되어야 하는가?)
- 실제 결과: (실제로 어떻게 되는가?)
```

## Step 3: 근본 원인 파악

"왜?" 를 최소 3번 반복한다.

```
왜 UI가 깨지는가? → 데이터가 undefined
왜 undefined인가? → 필터링 조건이 잘못됨
왜 필터링이 잘못되었는가? → slug 비교 시 대소문자 불일치
→ 근본 원인: slug 정규화 누락
```

## Step 4: 근본 원인 수정

- 증상이 아닌 근본 원인을 수정한다
- 수정 후 재현 시나리오로 검증한다
- 의미 있는 버그는 `/troubleshoot`로 문서화한다

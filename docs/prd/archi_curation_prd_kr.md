- **MVP 핵심 모듈**: Archi Curation · Map
- **제품 유형**: 큐레이터 중심 아카이빙 웹 경험

- **상태**: Draft
- **최종 업데이트**: 2026-03-11

## 1. 제품 개요

Archi Curation은 건축, 디자인, 가구, 인테리어 숍, 이벤트 정보를 아카이빙하고 큐레이션하는 웹 기반 프로젝트다.

MVP는 **Archi Curation · Map**으로 시작한다. 이 첫 번째 모듈은 건축물과 건축가를 지도 중심으로 탐색할 수 있는 경험에 집중한다. 이후에는 가구, 인테리어 관련 숍, 건축/디자인 이벤트까지 확장 가능한 구조를 갖는다.

이 서비스는 사용자 생성형 플랫폼이 아니다. 핵심 운영 모델은 다음과 같다.

- 큐레이터가 데이터를 수집하고 구조화한다.
- 사이트는 이를 2D/3D 기반의 시각적 인터랙션으로 보여준다.
- 방문자는 큐레이션된 장소들을 탐색하고, 선택하고, 다이어그램화된 결과물을 감상하거나 이미지로 추출한다.

즉, 핵심 가치는 **유저 계정 기반의 생산성 기능**이 아니라,  
**큐레이션 + 시각적 흥미 + 인터랙션 경험**에 있다.

---

## 2. 제품 비전

건축과 디자인 레퍼런스를 단순히 검색하는 것을 넘어,  
**공간적으로 느끼고, 그래픽적으로 경험할 수 있는 디지털 큐레이션 공간**을 만든다.

Archi Curation은 다음과 같은 인상을 주어야 한다.

- 개인이 운영하는 정제된 아카이브
- 디자인 감도가 있는 레퍼런스 라이브러리
- 지도 기반 탐색 도구
- 선택한 장소를 그래픽 결과물로 전환해볼 수 있는 시각적 놀이터

---

## 3. 문제 정의

건축과 디자인에 관심 있는 사람들은 보통 정보를 여러 곳에서 따로 모아야 한다.

- 지도 서비스
- 블로그
- 기관 웹사이트
- SNS
- 위키피디아
- 사진 아카이브
- 전시/행사 페이지

이 과정에서 다음과 같은 문제가 생긴다.

1. **정보가 지나치게 분산되어 있다.**
2. **기존 아카이브 형식은 공간적 이해가 약하다.**
3. **단순 리스트형 도구는 시각적 흥미가 낮다.**
4. **나만의 큐레이션 결과물을 만들려면 별도의 그래픽 작업이 필요하다.**
5. **1인 큐레이션 운영 시, 데이터 정리가 쉽게 어지러워진다.**

Archi Curation은 이를 다음 방식으로 해결한다.

- 큐레이터가 직접 관리하는 구조화된 데이터
- 지도 중심의 탐색
- 선택을 기반으로 한 2D/3D 시각화
- 이미지로 추출 가능한 결과물

---

## 4. 제품 목표

### 4.1 핵심 목표
- 큐레이터가 지속적으로 축적할 수 있는 개인 아카이브 시스템을 만든다.
- 빠르게 MVP를 출시해 건축 지도 탐색 경험을 구현한다.
- 수집한 데이터를 2D/3D 시각적 인터랙션으로 확장한다.
- 방문자가 선택한 장소들을 기반으로 “나만의 지도/큐레이션 이미지”를 만들어볼 수 있게 한다.

### 4.2 부가 목표
- 이후 숍, 가구, 이벤트 등으로 자연스럽게 확장 가능한 구조를 갖춘다.
- 포트폴리오 사이트와 연결 가능한 프로젝트 자산으로 활용한다.
- 스프레드시트 기반의 간편한 콘텐츠 운영 파이프라인을 만든다.

### 4.3 MVP 비목표
- 사용자 제출형 아카이브
- 커뮤니티 기능
- 계정/로그인 중심 기능
- 피드형 소셜 기능
- 복잡한 경로 최적화
- 무거운 3D 도시 시뮬레이션

---

## 5. 제품 원칙

1. **큐레이터 중심**  
   대중 참여보다 일관된 편집 관점이 품질을 만든다.

2. **과한 기능보다 시각 경험**  
   사이트는 데이터베이스 툴처럼 느껴지기보다, 탐색하고 감상하는 디자인 오브젝트처럼 느껴져야 한다.

3. **운영은 가볍고 간단하게**  
   큐레이터가 부담 없이 데이터를 쌓고 수정할 수 있어야 한다.

4. **인터랙션은 재미 요소**  
   지도 선택, 다이어그램 전환, 이미지 추출 경험이 사이트의 감정적 인상을 만든다.

5. **확장 가능한 구조**  
   MVP는 건축부터 시작하지만, 구조는 숍/이벤트까지 고려해 설계한다.

## 6. 타깃 사용자

### 6.1 주요 타깃

#### A. 건축/디자인에 관심 있는 방문자
- 도시, 건축물, 공간, 디자인 문화를 좋아함
- 시각적이고 탐색적인 인터페이스를 선호함
- 깊은 논문형 자료보다는 감도 있는 레퍼런스를 원함

#### B. 디자인 감도가 있는 여행자
- 여행 전후로 건축물, 숍, 전시를 찾아봄
- 공간이 도시 안에서 어떻게 배치되어 있는지 보고 싶어 함
- 단순 정보보다 “내가 발견한 것” 같은 인상을 남기고 싶어 함

#### C. 학생 / 초기 크리에이터
- 건축, 인테리어, 가구, 전시 레퍼런스를 모으는 데 관심이 있음
- 일반적인 검색보다 큐레이션된 아카이브를 선호함

### 6.2 보조 타깃

#### D. 포트폴리오 리뷰어 / 채용 담당자
- 이 사이트를 통해 제품 기획력, 프론트엔드 구현력, 시각 디자인 감도, 큐레이션 관점을 함께 확인할 수 있음

---

## 7. 핵심 사용자 경험

MVP는 아래 흐름을 지원해야 한다.

1. 방문자가 사이트에 들어온다.
2. 지도 / 리스트 / 그리드 뷰를 통해 큐레이션된 건축 콘텐츠를 탐색한다.
3. 흥미로운 장소를 여러 개 선택한다.
4. 선택한 지점들이 다이어그램/그래픽 형태로 재구성된다.
5. 결과물을 이미지로 추출한다.
6. 단순 정보 탐색을 넘어, 시각적으로 기억에 남는 경험을 얻는다.

이 사이트는 “정보를 정리해주는 서비스”라기보다  
**큐레이션된 세계를 경험하게 하는 웹 공간**이어야 한다.

## 10. 기능 요구사항

## 10.1 홈

### 목적
브랜드 톤과 사이트의 성격을 첫 화면에서 전달한다.

### 요구사항
- 그리드 기반의 강한 히어로 섹션
- 가벼운 3D 또는 라인 기반 모션
- 다음 진입점 제공
  - Map
  - Buildings
  - Architects
  - Featured city / featured curation
- 큐레이터 중심 아카이브라는 메시지 전달

### 성공 기준
- 사용자가 몇 초 안에 “이건 큐레이션된 건축/디자인 아카이브다”를 이해한다.
- 첫 화면만으로도 포트폴리오적 인상이 난다.

---

## 10.2 Map 모듈

### 목적
공간적으로 장소를 발견하게 한다.

### 요구사항
- 인터랙티브 지도
- 큐레이션된 엔티티 핀 표시 (MVP는 building 중심)
- 핀 클릭 시 프리뷰 카드 / 상세 진입
- 도시 필터
- 태그 / 스타일 필터
- 다중 지점 선택 가능

### 있으면 좋은 기능
- 핀 클러스터링
- 지도 옆 미니 리스트 패널

### 성공 기준
- 사용자가 도시 안에서 흥미로운 장소를 빠르게 훑을 수 있다.
- 여러 장소를 선택하는 경험이 직관적이다.

## 10.3 건축물 아카이브

### 목적
건축물 중심의 구조화된 레퍼런스 아카이브를 제공한다.

### 요구사항
- 건축물 리스트 / 그리드
- 상세 페이지에 다음 정보 포함
  - 이름
  - 도시
  - 국가
  - 연도
  - 건축가
  - 설명
  - 좌표
  - 출처 링크
  - 이미지 크레딧 / 라이선스
- 관련 항목 추천
- 태그 기반 내비게이션

### 성공 기준
- 상세 페이지가 정보와 시각적 여백의 균형을 가진다.
- 레퍼런스 아카이브이면서도 감도가 느껴진다.

---

## 10.4 건축가 아카이브

### 목적
건축물과 사람 사이의 맥락을 연결한다.

### 요구사항
- 건축가 리스트
- 건축가 상세 페이지에 다음 정보 포함
  - 이름
  - 국적
  - 생몰년
  - 짧은 소개
  - 대표작
  - 관련 건축물 링크

### 성공 기준
- 방문자가 사람과 장소를 자연스럽게 오갈 수 있다.

---

## 10.5 Diagram Mode / 시각화 결과 생성

### 목적
선택한 장소를 그래픽 경험으로 전환한다.

### 핵심 아이디어
사용자가 지도에서 여러 지점을 선택하면, 그 선택들이 하나의 다이어그램 또는 그래픽 구성으로 변환된다.

### 요구사항
- 여러 지도 포인트를 선택 가능
- “Generate Diagram” 액션 제공
- 결과물에는 다음이 포함될 수 있음
  - 포인트 / 마커
  - 연결선
  - 라벨 또는 번호
  - 브랜드 아이덴티티가 반영된 그리드 / 추상화된 지리 프레임
- 결과를 PNG/WebP로 추출 가능

### 비주얼 방향
- 실제 지도 스크린샷보다 추상적이어야 함
- 그리드, 선, 도면 감각이 살아 있어야 함
- “실용적 경로지도”보다 “개인 큐레이션 프린트”처럼 느껴져야 함

### 성공 기준
- 결과물이 저장하고 싶을 정도로 예쁘다.
- 단순한 지도 인터랙션보다 한 단계 더 기억에 남는다.

## 11. AI 경험 방향

이 제품에서 AI는 생산성 어시스턴트처럼 느껴지면 안 된다.  
**큐레이션 해석 레이어**처럼 작동해야 한다.

### 가능한 향후 AI 기능
- 선택한 장소와 비슷한 지점 추천
- 테마별 큐레이션 제안
  - 브루탈리즘 루트
  - 모더니즘 루트
  - 목재/인테리어/가구 중심 루트
- 선택한 세트에 대한 짧은 큐레이터 캡션 생성
- 다이어그램 결과물의 제목 제안

### 원칙
AI는 사이트의 주인공이 아니라,  
큐레이션된 탐색 경험을 조금 더 풍부하게 만드는 도구여야 한다.

## 13. 콘텐츠 운영 방식

## 13.1 권장 입력 방식
MVP에서는 **스프레드시트 기반 운영**을 권장한다.

### 권장 구성
- Google Sheets를 메인 아카이브 DB처럼 사용
- Google Form으로 모바일 빠른 입력 가능
- 시트 탭 분리
  - Curated
  - Cities
  - Architects
  - Images

### 이유
- 입력 마찰이 낮다.
- 수동 큐레이션에 적합하다.
- 모바일 입력이 쉽다.
- 1인 운영에 부담이 적다.
- 이후 JSON / DB로 변환하기 쉽다.

## 13.2 퍼블리싱 흐름
1. 큐레이터가 스프레드시트에 데이터 입력/수정
2. `published` 상태인 행만 사이트 반영 대상
3. 스크립트가 스프레드시트/CSV를 JSON으로 변환
4. 사이트가 JSON을 직접 읽거나 Supabase에 동기화

## 13.3 검증 필요 항목
- slug 중복 여부
- 좌표 유효성
- 라이선스 허용 목록 여부
- 필수 필드 누락 여부
- 태그 표준화

## 18. 리스크

### 18.1 운영 리스크
수동 큐레이션이 지나치게 복잡하면 운영이 느려질 수 있다.

**대응**: 입력 구조를 최소화하고 표준화한다.

### 18.2 저작권 리스크
이미지 출처 관리가 약하면 법적/윤리적 문제가 생길 수 있다.

**대응**: 이미지 메타데이터가 완전한 항목만 공개한다.

### 18.3 성능 리스크
지도 + 3D + 이미지 추출 기능이 무거워질 수 있다.

**대응**: MVP에서는 시각 효과를 추상적이고 가볍게 제한한다.

### 18.4 범위 리스크
초기에 카테고리를 너무 넓히면 런칭이 흐려질 수 있다.

**대응**: 첫 출시에서는 architecture map에 집중하되, 구조만 확장 가능하게 설계한다.

## 21. 최종 요약

Archi Curation은 큐레이터가 직접 운영하는 아카이빙 웹 프로젝트이며,  
첫 단계에서는 건축 중심의 지도 탐색 경험으로 시작한다.

이 프로젝트의 차별점은 단순한 정보 정리가 아니라 다음의 결합에 있다.

- 편집된 관점
- 지도 기반 발견
- 건축적인 시각 언어
- 선택을 그래픽 결과물로 바꾸는 다이어그램형 인터랙션

MVP의 성공은 다음을 만족하는 것이다.

- 큐레이터가 데이터를 쉽게 쌓을 수 있고
- 방문자가 직관적으로 탐색할 수 있으며
- 사이트 자체가 하나의 인상적인 디자인 결과물처럼 느껴진다

# Interview Transcript — Archi Curation Implementation Plan

## Q1. 현재 구현된 기능 중 개선이 필요한 부분은?
**Answer**: 지도 필터링 부재, 데이터 관리

## Q2. 이번 계획의 범위는?
**Answer**: Phase 2 + 3 (Core Archive 완성 + Visual Interaction 개선)

## Q3. 데이터 파이프라인 구성 방향은?
**Answer**: 당장 개발하는 데는 JSON 연결이 편한데, 로컬 기반이라 결국 구글 시트로 마이그레이션해서 여러 곳에서 접근 가능하도록 해야 할 듯. → JSON 유지 + Google Sheets 마이그레이션 경로 준비

## Q4. 지도 필터링 UI 구성 방향은?
**Answer**: 필터가 이미 있는 것 같은데 한번 확인해봐. → 기존 필터 구현 상태 확인 필요 (리서치에서 map 페이지에 filter UI가 없음을 확인, buildings 리스트에는 태그 필터가 있음)

## Q5. 다이어그램 결과물의 비주얼 방향은?
**Answer**: 지금도 좋긴 한데, 뭔가 1차 다이어그램에서 2차적으로 어떤 그래픽으로 전환되는 게 있어도 재밌을 것 같아. 근데 기획 방향성에 대해서 고민이 필요해.

## Q6. 다이어그램 2차 그래픽 전환 방향은?
**Answer**: 도면풍 그리드 레이아웃, 포스터풍 프린트 — 둘 다 테스트해보고 싶어.

## Q7. 시드 데이터 확장 계획은?
**Answer**: 50개 이상이 목표인데, 이건 차차 늘려갈 거야. → 기능 우선, 데이터는 점진적 확장

## Q8. 이미지 추출 우선순위는?
**Answer**: 현재 PNG 충분

## Q9. Shops/Events 모듈은?
**Answer**: 기본 UI 구축

## Q10. 홈 페이지 개선 방향은?
**Answer**: 전면 재디자인 (포트폴리오적 인상 강화)

## Q11. 성능/기술적 관심사는?
**Answer**: 클러스터링 + 이미지 최적화 둘 다 필요

## Q12. 추가 요청 (인터뷰 중)
**Answer**: 이미지 레이아웃 시프트 생기지 않도록 하는 것도 포함해달라

## Interview Summary

### 범위
- PRD Phase 2 (Core Archive) + Phase 3 (Visual Interaction)

### 핵심 구현 항목
1. **지도 필터링 UI** — 도시/태그 필터 추가 (기존 상태 확인 후)
2. **다이어그램 개선** — 현재 유지 + 2차 그래픽 전환 탐색 (도면풍, 포스터풍)
3. **홈 전면 재디자인** — 포트폴리오적 인상, 그리드 히어로, 라인 모션
4. **Shops/Events 기본 UI** — 리스트 페이지 수준
5. **데이터 파이프라인** — JSON 유지 + Google Sheets 마이그레이션 경로
6. **성능** — 마커 클러스터링 + 이미지 최적화 (레이아웃 시프트 방지 포함)

### 보류 항목
- 다이어그램 2차 그래픽 전환의 구체적 기획 (방향성 고민 필요)
- Export 포맷 확장 (현재 PNG 충분)
- Supabase 전환 (추후)
